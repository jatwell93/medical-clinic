3. **Code cell (§6 clinic_pnl function)** — implement `clinic_pnl(a: dict) -> dict` EXACTLY as specified in 04-RESEARCH.md "clinic_pnl Function Skeleton", with these concrete structural rules:
   - `annual_consults = a["n_gp_fte"] * a["gp_fte_consults_per_yr"] * a["utilisation"]` (capacity-driven, D-03)
   - `item_mix = a["item_mix"]` — dict of {item_number: {"pct": float, "rebate": float, "private_fee": float}}
   - `weighted_rebate = sum(v["pct"] * v["rebate"] for v in item_mix.values())`
   - `weighted_private_fee = sum(v["pct"] * v["private_fee"] for v in item_mix.values())`
   - `avg_rev_per_consult = a["bulk_bill_share"] * weighted_rebate + (1 - a["bulk_bill_share"]) * weighted_private_fee`
   - `gp_billings = annual_consults * avg_rev_per_consult` — INFORMATIONAL top line (Pitfall 11 guardrail)
   - `practice_revenue_gp = gp_billings * a["gp_revenue_share"]` — THE revenue line (service-fee %)
   - `allied_consults = a["n_allied_fte"] * a["allied_consults_per_yr"] * a["utilisation"]`
   - `allied_billings = allied_consults * a["allied_avg_rev_per_consult"]`
   - `practice_revenue_allied = allied_billings * a["gp_revenue_share"]` (D-02: same service-fee % split)
   - `practice_revenue = practice_revenue_gp + practice_revenue_allied`
   - `staffing_costs = a["nurse_fte"] * a["nurse_salary_yr"] * a["oncost_multiplier"] + a["reception_fte"] * a["reception_salary_yr"] * a["oncost_multiplier"] + a["manager_fte"] * a["manager_salary_yr"] * a["oncost_multiplier"]`
   - `costs = a["rent_yr"] + staffing_costs + a["insurance_yr"] + a["consumables_yr"] + a["admin_it_utilities_yr"]`
   - `ebitda = practice_revenue - costs`
   - `margin = ebitda / practice_revenue if practice_revenue else 0`
   - Return dict with keys: `annual_consults`, `gp_billings`, `practice_revenue`, `practice_revenue_gp`, `practice_revenue_allied`, `staffing_costs`, `total_costs`, `ebitda`, `margin`, `service_fee_pct` (= `a["gp_revenue_share"]`)
   - **CRITICAL: GP payments are NOT a cost line.** The GP's 65% share is already deducted at the revenue line (billings × service_fee_% = practice revenue). Do NOT add a `gp_payments` cost. Double-counting is the other common Pitfall 11 error variant.
   - **CRITICAL: No BBPIP, BBI, or bulk billing incentive in this function.** Base case is 70/30 mixed billing only. BBPIP is Phase 5 (FIN-06, Pitfall 7).
