## Fixed Issues

### WR-05: `check_dataflow_exists` returns False for valid-XML non-SDMX responses

**Files modified:** scripts/extend_v2_notebook_demographics.py, Johnston_St_v2.ipynb
**Commit:** 4b72c2d

**Applied fix:** Added a guard after both `flow_ids` searches (namespaced + namespace-agnostic fallback): if `flow_ids` is still empty after both searches, the function prints a warning and returns `True` (safe default) instead of `flow_id in []` = `False`. This ensures that valid-XML non-SDMX responses (e.g., CDN error pages, maintenance pages in XHTML) trigger the safe default path — the fetch is attempted and downstream try/except fallbacks handle failures. This aligns the function's behavior with its docstring ("Safe default: returns True on any failure") and the no-hard-fail principle.

### WR-06: Age-band prefix matching misses 5+ known ABS G04 column name variants

**Files modified:** scripts/extend_v2_notebook_demographics.py, Johnston_St_v2.ipynb
**Commit:** 361ce96

**Applied fix:** Replaced the case-sensitive `startswith()` + exact-match set with a case-insensitive regex: `re.compile(r"age(?:_yr)?_(65|70|75|80|85)", re.IGNORECASE)`. This single pattern covers all 10 known ABS G04 column variants:
- `Age_yr_65_69`, `Age_yr_85ov` (API CSV-with-labels)
- `Age_65_69`, `Age_85plus` (DataPack exact)
- `Age_85_over`, `Age_65_69_yr`, `Age_70_74_yr` (DataPack `_over`/`_yr` suffix variants)
- `AGE_YR_65_69`, `AGE_65_69`, `AGE_85_OVER` (uppercase variants)

Verified with a test script: all 10 65+ variants match, and 5 non-65+ columns (`Age_yr_0_4`, `Age_yr_5_9`, `Age_yr_60_64`, `Total_P_P`, `POA_CODE21`) are correctly excluded.

### WR-07: `fetch_g04_fallback` returns `pct_65plus: 0` — misleading (should be NaN)

**Files modified:** scripts/extend_v2_notebook_demographics.py, Johnston_St_v2.ipynb
**Commit:** ff9d132

**Applied fix:** Changed `"pct_65plus": 0` to `"pct_65plus": float("nan")` in the `fetch_g04_fallback()` stub DataFrame. Zero percent 65+ is not a valid demographic value for any postcode — NaN ensures the peer table and 2x2 chart show "N/A" rather than a misleading 0% bar. Consistent with the WR-02 fix pattern (zero is not a valid population/percentage).

### WR-08: §3.5 peer_table crashes with KeyError on `pd.DataFrame()` "none" fallback

**Files modified:** scripts/extend_v2_notebook_demographics.py, Johnston_St_v2.ipynb
**Commit:** 0d481b8

**Applied fix:** Applied option (a) from the recommendation — return empty-schema DataFrames with correct columns in the "none" fallback case (GCP file also missing), consistent with the WR-02 "fallback" case pattern. Specifically:
- `fetch_g01_poa()` "none" path: `pd.DataFrame(columns=["POA_CODE21", "Total_P_P", "M_P", "F_P", "Total_Dwll_D"])` instead of `pd.DataFrame()`
- `fetch_g02_poa()` "none" path: `pd.DataFrame(columns=["POA_CODE21", "Median_age_persons", "Median_tot_hshld_inc_weekly"])` instead of `pd.DataFrame()`

This prevents the `KeyError: "['POA_CODE21', 'Total_P_P'] not in index"` crash at §3.5 peer_table construction in the double-failure scenario (ABS API down + local GCP file missing). The empty-schema DataFrames have the correct columns for safe downstream merges, producing an empty peer table instead of a crash.

### WR-09: ABS API CSV format assumption (wide vs long) unverified — may cause silent failures

**Files modified:** scripts/extend_v2_notebook_demographics.py, Johnston_St_v2.ipynb
**Commit:** f7e4a6f

**Applied fix:** Added a `tidy_abs_csv(df)` helper function that detects long-format ABS CSV responses (presence of `OBS_VALUE` and `MEASURE` columns) and pivots them to wide format using `pivot_table(index=index_cols, columns="MEASURE", values="OBS_VALUE", aggfunc="first").reset_index()`. The helper is called inside `fetch_abs_csv()` after `pd.read_csv()`, so all fetch functions (G01, G02, G04, SA1) automatically benefit. If the data is already in wide format, the helper returns it unchanged (no-op). A print warning is emitted when long format is detected and pivoted, providing teaching transparency.

Tested with simulated long-format and wide-format DataFrames — long format correctly pivoted to one row per geography with measures as columns; wide format correctly unchanged.

**Status:** fixed: requires human verification — the long-format detection logic is based on the SDMX standard `OBS_VALUE`/`MEASURE` column names. The actual ABS API CSV-with-labels column names should be verified by executing the notebook against the live API at the earliest opportunity.
